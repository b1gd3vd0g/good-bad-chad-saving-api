player

Get signed in player account

Fetch a single player from the database using an authentication token (safe information only).

header Parameters
Authorization
required
string

Contains the prefix "BEARER " followed by a JSON web token provided by logging in at "POST /player/login".

Responses

Production server

https://goodbadchad.bigdevdog.com:6900/player

Response samples

  • 200
  • 401
  • 500
Content type
application/json
{
  • "player_id": "string",
  • "username": "string",
  • "email": "string",
  • "created": "string"
}
